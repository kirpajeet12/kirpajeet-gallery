# Release Notes — 2026-06-15

## kirpajeet-gallery v1.0

Personal photo gallery built by Kirpajeet Singh Gill.

## What We Built

### Core Stack
- **Next.js 14** (App Router) — frontend framework
- **AWS Amplify Gen 2** (CDK-based) — backend infrastructure as code
- **AWS S3** — photo and music storage
- **AWS DynamoDB** (via AppSync GraphQL) — photo metadata (caption, musicKey, order)
- **AWS Cognito** — authentication (owner-only upload/edit/delete; public read)

### Features
- Masonry photo grid (CSS columns, natural aspect ratios)
- Instagram-style story popup — card style, white frosted glass overlay, progress bars
- Auto-advance stories (6 seconds per slide)
- Tap left/right zones to navigate slides
- Deezer music search — search any song, attach 30-second preview to a photo
- Music badge (♪) on gallery tiles that have music
- Music auto-plays when story opens, tap label to pause/resume
- Upload modal — photo + optional music + caption
- Edit modal — change caption or swap music on any existing photo
- Delete with confirmation — removes photo and music from S3 + DB record
- Owner login (Cognito) — edit/delete buttons hidden from public visitors

## Key Lessons Learned (Technical)

### Auth Modes
- Amplify Gen 2 defaults to `identityPool` auth for data operations
- `allow.authenticated()` in the schema requires `userPool` auth on the client
- Fix: pass `{ authMode: 'userPool' }` on every Memory.create / update / delete / list call
- Public visitors use `identityPool` for reads (guest access)

### DynamoDB / AppSync Field Types
- `a.integer()` is 32-bit — max 2,147,483,647
- `Date.now()` = ~1.75 trillion — overflows Int
- Fix: `Math.floor(Date.now() / 1000)` for Unix timestamps as order field
- Never add a field to the schema locally without redeploying sandbox first

### S3 + Music Playback
- Deezer CDN blocks server-side streaming requests (detects non-browser User-Agent / IP)
- Browser `<audio>` can play Deezer preview URLs directly (browser headers pass)
- Best pattern: fetch Deezer preview server-side as a one-time download → upload to S3 → store S3 key
- S3 presigned URLs play 100% reliably with no CDN restrictions
- Our music proxy (`/api/music-proxy`) works for one-time download but not streaming

### Node Version
- Node 25.x broke `ampx sandbox` — `localStorage.getItem is not a function`
- Fix: downgrade to Node 20 LTS via `brew install node@20` + PATH update

## Architecture

```
Browser (Next.js)
    │
    ├── AWS Amplify Storage (S3)
    │     photos/   — uploaded images
    │     music/    — Deezer previews downloaded at upload time
    │
    ├── AppSync GraphQL API
    │     Memory model: { imageKey, musicKey, caption, order }
    │     Auth: identityPool (read) + userPool (write)
    │
    └── Cognito User Pool
          Owner account only — kirpajeets6@gmail.com
```
