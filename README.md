# Kirpajeet Singh Gill — Photo + Music Gallery

A personal gallery webpage. Visitors browse photos; tapping any photo plays a
full-screen "story" slideshow with the music you attached to each photo. Only
**you** (signed in) can upload. Built on **Next.js 14 (App Router) + AWS Amplify Gen 2**.

## What's inside

```
amplify/
  auth/resource.ts      # email login (you only)
  data/resource.ts      # Memory model: imageKey, musicKey, caption, order
  storage/resource.ts   # S3 bucket -> photos/ and music/ folders
  backend.ts
app/
  page.tsx              # gallery grid + login + upload + story
  components/
    UploadModal.tsx     # pick photo + music + caption
    StoryViewer.tsx     # fullscreen slideshow, music plays per photo
  ConfigureAmplify.tsx
  layout.tsx
  globals.css
lib/client.ts
```

## Run it locally (5 steps)

You need: Node 18.17+, npm 9+, git, and an AWS account with local credentials
(`aws configure` with a profile that has `AmplifyBackendDeployFullAccess`).

```bash
# 1. install
npm install

# 2. start your personal cloud sandbox (creates S3 + DynamoDB + Cognito,
#    and generates amplify_outputs.json automatically — takes ~5 min first run)
npx ampx sandbox

# 3. in a SECOND terminal, run the site
npm run dev
# open http://localhost:3000
```

First time: click **Login → Create Account**, confirm the email code, and you're
the owner. Then **+ Add moment** to upload a photo, optional music, and caption.

## Deploy it live (Amplify Hosting)

```bash
# 1. push to a new GitHub repo
git init && git add . && git commit -m "gallery"
git branch -M main
git remote add origin <your-repo-url>
git push -u origin main
```

Then in the **AWS Amplify console**:
1. Create new app → **Deploy with GitHub** → pick this repo + `main` branch.
2. Amplify auto-detects Next.js and builds the backend (auth/data/storage) +
   frontend together. Click **Save and deploy**.
3. ~5 min later you get a live URL (e.g. `https://main.xxxx.amplifyapp.com`).
   You can attach a custom domain (kirpajeetsinghgill.com) in the Hosting tab.

## Notes
- `amplify_outputs.json` is generated by the sandbox / Amplify build — it's
  gitignored on purpose; the cloud build regenerates it.
- Photos and music live in S3; metadata in DynamoDB. Visitors get read-only
  signed URLs. Uploads/deletes require sign-in.
- Story auto-advances every 6s; tap left/right to move, × to close.
